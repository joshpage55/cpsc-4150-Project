Backend Proxy & Secret Handling (M2)

Team: CPSC 4150 ReadRight  |  Project: cpsc4150-readright

1. Problem

The Flutter app must never embed LLM or STT API keys. M2 requires generative-AI calls through a thin backend proxy. M2 proves this with the story spike; M3 will move Deepgram STT behind the same pattern.

2. Architecture

Flutter app → HTTPS (no key) → Cloud Function generateStorySpike → GEMINI_API_KEY (secret) → Gemini mini-tier API
LayerResponsibility
FlutterSends Dolch word list; displays story
Cloud FunctionPrompt assembly, model allowlist, calls Gemini
Secret storefirebase functions:secrets:set GEMINI_API_KEY
GitHubSource only; .env gitignored

4. Dolch prompt design

Proxy lists allowed Dolch words, caps length (80–120 words), targets grade 1–2 reading level, returns plain text. See functions/storyPrompt.js.

5. Model guardrail

Mini-tier only (GEMINI_MODEL=gemini-2.0-flash-lite). Function rejects models outside allowlist.

6. GitHub safety

.gitignore blocks .env; .env.example has names only; review PRs for leaked keys; rotate if committed.
